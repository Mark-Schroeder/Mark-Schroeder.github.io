Project Idea



The Automatic Picture Printer


Overview

The Automatic picture printer will be a Windows Form Application that gives a user three options (3 buttons). The options would be: “Cat”, “Dog” and “NASA”. When the user clicks on one of these options, the program will use Application Programming Interfaces (API) to fetch images of a cat, dog or of space (depending on which button they select), from the internet. This (some-what random) image would then directly print to the user’s default printer. This would be a fun surprise when it comes out of the printer as the user does not fully know what to expect, giving an element of surprise!


Motivation

Weiss et al. (2012) found that generally, people have an interest in the way dogs look. This in addition to the element of surprise will hopefully make an exiting application that users will love. Additionally, this application would allow the user to see something cute and or inspiring in the real world with just one click of a button. By doing this, people who use this application daily, will hopefully have a smile on their face when they see an image of an adorable puppy, or a deep sense of awe when they see an image of deep space. This has the potential to bring positivity to some one’s day, and hopefully excite them about space technology and exploration.


Description

The application will be a Windows Form Application written in C# that would be available to the public through a download link on a website or the Microsoft store. As discussed earlier, when the user opens the application, they will have options for what sort of image they want to receive, and when they select a button, the app will search the web for an appropriate image. The application will search through existing databases with the images and any other relevant information, request it, decode it (if needed) and send it to the user’s default printer. The application would give the user a message regarding whether the execution of the program was successful or not. In the event of the application failing, the app should parse an error message to the user through a message box or a printout onto the existing window. A possible drawback of the app is that many will want to have more options to choose from. This can be combatted by being open to feedback from users and by asking for reviews, for example, doing surveys, etc.

Tools and Technology

To build this I will use Visual Studio to create the Windows Form App. One of the reasons I like Visual Studio is because I find it easy to prototype different layouts of forms. This gives me flexibility when designing, creating, and rapidly iterating through ideas for the product. I can easily place and program each button and change functionality of each attribute very quickly. Beyond the application that the user will see will be code that executes the user’s input. The language I have chosen to use for this is C# (pronounced ‘C sharp’), ‘a modern, object-oriented, and type-safe programming language’ (Microsoft 2022, para. 1). Because C# is regularly updated to support up and coming coding and design practices (Microsoft 2022), I am confident that the Automatic Picture Printer will be easily maintained and kept up to date with modern systems. C# can also ‘help create robust and durable applications’ (Microsoft 2022, para. 3) which gives me further confidence in its capabilities.

Cool. But how will it actually work? How does it get the image from the internet? In a C# application you can make HTTP requests to the GitHub REST service (Microsoft 2021). This would allow you to get JavaScript Object Notation (JSON) data directly into your program. ‘JSON is a lightweight data-interchange format’ (json.org n.d.). After you deserialize the response (JSON data) from the repository, you can use that object for what you please within your program, for example, display the data in your console. This is useful if we want to gather data about what we are printing, or other useful data, for example, facts about what we are printing.

Another method of requesting data from a website using a URI (Uniform Resource Identifier) is by using the WebClient function (Microsoft 2022). The WebClient.DownloadFile method simply ‘downloads the resource with the specified URI to a local file’ (Microsoft 2022) which can then be used by the Automatic Picture Printer to send to the user’s printer. But where are the images coming from? You may ask. Cat images can be fetched here, dog images can be fetched here, and NASA images can be fetched here.

Skills required

To complete this project, I will need to be able to code in C#, do basic design, understand how to use APIs, understand JSON and other file/data types, and know how to put all these things together to produce the finished application. The creation of this product will also require much patience as these things can get very frustrating when they are not working as intended. Research skill will also be needed when I am unsure of how to go about parts of the development. This research can be done through reading documentation or interacting with coding communities on forums.

Outcome

If the Automatic Picture Printer project was to be made, users would be able to have a fun and surprising product that can hopefully bring joy to their day. When they select a button, they will be greeted, in the real world, with a somewhat randomized image from the internet.
